Reflect on how confident you feel using each of these problem-solving techniques/processes:

▧ Pseudocode:

Pseudocode acts as the bridge between our brain and computer’s code executor. It allows us to plan instructions which follow a logical pattern, without including all of the technical details. Pseudocode is a great way of getting started with software programming as a beginner. we won’t have to overwhelm our brain with coding syntax.
It is also much easier and more convenient to break our code into small steps and work on small steps to find the code we need.

▧ Trying something:

yeah.. sometimes something pops into my head and i try to test it to see if it works

▧ Rubber ducky method:

I mostly explain the code to someone else, when I find an error in my coding (even if there's no one around me :)) it helps me see the code I wrote better .
The first thing I can solve with this method is that I can find typos, missing commas or semicolons.

▧ Reading error messages:

it's a good way to find the error, but I think it's not enough.

▧ Console.logging:

It's a very useful way to be sure about the code, and always I use it

▧ Googling:

The first thing I do when I want to write code is to google. and try to find out more about it.
mostly I google the question and i can find some similar ones, then I try to write my code based on them and in the end I use console.log(); to be sure about my answer.

▧ Asking your peers and coaches for help:

I ask my peers or coaches , If I can't solve the problem with all the above ways.
